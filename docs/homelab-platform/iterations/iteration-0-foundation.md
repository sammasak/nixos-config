### 0.1 NixOS Base Configuration

**Goal:** Configure NixOS modules for k3s.

**Tasks:**
- [x] Create `modules/homelab/k3s/server.nix` for k3s server
- [x] Create `modules/homelab/k3s/agent.nix` for k3s agents
- [x] Configure networking (flannel CNI)
- [x] Set up firewall rules

**NixOS Module (Server):**

```nix
# modules/homelab/k3s/server.nix
{ config, lib, pkgs, ... }:
{
  options.homelab.k3s.server = {
    enable = lib.mkEnableOption "k3s server";
    clusterToken = lib.mkOption {
      type = lib.types.str;
      description = "Cluster token for node authentication";
    };
  };

  config = lib.mkIf config.homelab.k3s.server.enable {
    services.k3s = {
      enable = true;
      role = "server";
      token = config.homelab.k3s.server.clusterToken;
      extraFlags = toString [
        "--disable=traefik"  # We'll use ingress-nginx
        "--disable=servicelb"
        "--flannel-backend=host-gw"
        "--write-kubeconfig-mode=644"
      ];
    };

    # Firewall rules for k3s
    networking.firewall = {
      allowedTCPPorts = [
        6443  # Kubernetes API
        2379  # etcd client
        2380  # etcd peer
        10250 # Kubelet
      ];
      allowedUDPPorts = [
        8472  # Flannel VXLAN
      ];
    };

    # Required packages
    environment.systemPackages = with pkgs; [
      kubectl
      kubernetes-helm
    ];
  };
}
```

**NixOS Module (Agent):**

```nix
# modules/homelab/k3s/agent.nix
{ config, lib, pkgs, ... }:
{
  options.homelab.k3s.agent = {
    enable = lib.mkEnableOption "k3s agent";
    serverAddr = lib.mkOption {
      type = lib.types.str;
      description = "k3s server address";
    };
    clusterToken = lib.mkOption {
      type = lib.types.str;
      description = "Cluster token for authentication";
    };
  };

  config = lib.mkIf config.homelab.k3s.agent.enable {
    services.k3s = {
      enable = true;
      role = "agent";
      serverAddr = config.homelab.k3s.agent.serverAddr;
      token = config.homelab.k3s.agent.clusterToken;
    };

    networking.firewall = {
      allowedTCPPorts = [ 10250 ];
      allowedUDPPorts = [ 8472 ];
    };
  };
}
```

### 0.2 Colmena Deployment

**Goal:** Configure multi-host deployment with Colmena.

**Tasks:**
- [ ] Create `colmena.nix` configuration
- [ ] Define host inventory
- [ ] Test deployment to target hosts
- [ ] Set up CI for deployment (optional)

**Colmena Configuration:**

```nix
# colmena.nix
{
  meta = {
    nixpkgs = import <nixpkgs> { };
    specialArgs = {
      # Shared arguments for all hosts
    };
  };

  defaults = { pkgs, ... }: {
    imports = [
      ./modules/homelab/k3s/server.nix
      ./modules/homelab/k3s/agent.nix
    ];

    # Common configuration
    nix.settings.experimental-features = [ "nix-command" "flakes" ];
  };

  # k3s server (primary)
  homelab-server = { name, nodes, ... }: {
    deployment = {
      targetHost = "192.168.1.10";
      targetUser = "root";
    };

    homelab.k3s.server = {
      enable = true;
      clusterToken = "my-cluster-token";  # Use sops-nix in practice
    };

    networking.hostName = "homelab-server";
  };

  # Additional agents can be added here
}
```

**Deployment Commands:**

```bash
# Deploy to all hosts
colmena apply

# Deploy to specific host
colmena apply --on homelab-server

# Build locally, deploy remotely
colmena build
colmena apply --on homelab-server
```

### 0.3 Flux GitOps Bootstrap

**Goal:** Install Flux and configure Git repository sync.

**Tasks:**
- [ ] Create GitHub repository (or use existing nixos-config)
- [ ] Bootstrap Flux into cluster
- [ ] Create initial Kustomization structure
- [ ] Verify reconciliation working

**Bootstrap Command:**

```bash
# Export GitHub token
export GITHUB_TOKEN=<your-token>

# Bootstrap Flux (points to separate homelab-gitops repo)
flux bootstrap github \
  --owner=<your-username> \
  --repository=homelab-gitops \
  --branch=main \
  --path=clusters/homelab \
  --personal
```

**Directory Structure (in homelab-gitops repo):**

```
clusters/homelab/
├── flux-system/           # Auto-generated by bootstrap
│   ├── gotk-components.yaml
│   ├── gotk-sync.yaml
│   └── kustomization.yaml
├── infra/        # Platform components
│   └── kustomization.yaml
├── sources/               # GitRepository, HelmRepository
│   └── kustomization.yaml
└── apps/                  # Application deployments
    └── kustomization.yaml
```

**Infrastructure Kustomization:**

```yaml
# clusters/homelab/infra/kustomization.yaml
apiVersion: kustomize.toolkit.fluxcd.io/v1
kind: Kustomization
metadata:
  name: infra
  namespace: flux-system
spec:
  interval: 10m
  sourceRef:
    kind: GitRepository
    name: flux-system
  path: ./clusters/homelab/infra
  prune: true
  wait: true
```

### 0.4 SOPS Secrets Integration

**Goal:** Encrypt secrets in Git, decrypt at runtime.

**Tasks:**
- [ ] Generate age key for cluster
- [ ] Configure sops-nix on NixOS hosts
- [ ] Configure Flux SOPS decryption
- [ ] Test encrypted secret deployment

**Generate Age Key:**

```bash
# Generate key
age-keygen -o age.agekey

# Get public key (for encryption)
cat age.agekey | grep "public key"

# Store private key securely (will be added to cluster)
```

**SOPS Configuration:**

```yaml
# .sops.yaml
creation_rules:
  - path_regex: clusters/homelab/.*\.yaml$
    encrypted_regex: "^(data|stringData)$"
    age: age1xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

**Flux SOPS Secret:**

```yaml
# clusters/homelab/flux-system/sops-age-secret.yaml
apiVersion: v1
kind: Secret
metadata:
  name: sops-age
  namespace: flux-system
stringData:
  age.agekey: |
    # AGE-SECRET-KEY-1XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
```

**Kustomization with SOPS:**

```yaml
apiVersion: kustomize.toolkit.fluxcd.io/v1
kind: Kustomization
metadata:
  name: apps
  namespace: flux-system
spec:
  interval: 10m
  sourceRef:
    kind: GitRepository
    name: flux-system
  path: ./clusters/homelab/apps
  prune: true
  decryption:
    provider: sops
    secretRef:
      name: sops-age
```

### 0.5 Sample Workload

**Goal:** Deploy a test application via GitOps.

**Tasks:**
- [ ] Create nginx deployment manifest
- [ ] Add to apps Kustomization
- [ ] Verify Flux deploys it
- [ ] Access via kubectl port-forward

**Sample Deployment:**

```yaml
# clusters/homelab/apps/test-nginx/deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: test-nginx
  namespace: default
spec:
  replicas: 1
  selector:
    matchLabels:
      app: test-nginx
  template:
    metadata:
      labels:
        app: test-nginx
    spec:
      containers:
        - name: nginx
          image: nginx:alpine
          ports:
            - containerPort: 80
---
apiVersion: v1
kind: Service
metadata:
  name: test-nginx
  namespace: default
spec:
  selector:
    app: test-nginx
  ports:
    - port: 80
      targetPort: 80
```

## Verification Steps

```bash
# 1. Check k3s is running (on server node)
sudo systemctl status k3s

# 2. Check nodes
kubectl get nodes
# Expected: All nodes in Ready state

# 3. Check Flux
flux check
# Expected: All checks pass

# 4. Check reconciliation
flux get kustomizations
# Expected: All kustomizations ready

# 5. Check test deployment
kubectl get pods -l app=test-nginx
# Expected: Pod running

# 6. Test secret decryption
kubectl get secret test-secret -o yaml
# Expected: Decrypted values in data field
```

## Troubleshooting

| Issue | Check | Solution |
|-------|-------|----------|
| k3s won't start | `journalctl -u k3s` | Check token, firewall |
| Nodes not joining | Network connectivity | Verify firewall ports |
| Flux not reconciling | `flux logs` | Check Git credentials |
| SOPS decryption fails | `flux logs -n flux-system` | Verify age key in secret |
